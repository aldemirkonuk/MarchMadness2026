# Debug Analytics Notebook

Create `notebooks/model_diagnostics.ipynb` -- a single Jupyter notebook with 6 sections focused on debugging and improving the model. All data comes from the existing pipeline; no new dependencies needed (matplotlib, seaborn, shap, plotly all installed).

## Data Loading (Cell 1)

Load teams, run Phase 1A + 1B, collect all parameters into a DataFrame. Reuse existing code:

```python
from src.data_loader import load_all_teams
from src.weights import CORE_WEIGHTS, ORIGINAL_WEIGHTS, PARAM_KEYS
```

Also load `data/results/team_parameters.csv` (already generated by `main.py`).

## Section 1: Parameter Correlation Heatmap

- **Tool**: seaborn `heatmap` on the 40-parameter correlation matrix
- **Purpose**: Find redundant parameters (e.g., is `adj_em` still correlated with `barthag`, `z_rating`, `ppg_margin`?). Identify clusters that could be merged or where weight is being wasted.
- **Key output**: 40x40 heatmap with hierarchical clustering (seaborn `clustermap`)

## Section 2: Weight Sensitivity Waterfall

- **Tool**: matplotlib horizontal bar chart
- **Purpose**: Visualize the optimizer's sensitivity analysis results -- which parameters matter most when perturbed +/-50%. Shows at a glance where to invest tuning effort.
- **Data**: Run `stage2_sensitivity()` from [src/weight_optimizer.py](src/weight_optimizer.py) on the current `CORE_WEIGHTS`

## Section 3: SHAP Feature Importance (XGBoost)

- **Tool**: `shap.TreeExplainer` + `shap.summary_plot`
- **Purpose**: See which features the XGBoost model (Phase 1B) actually uses vs what we *think* matters. Compare SHAP importance ranking to our weight ranking -- large disagreements = tuning opportunities.
- **Data**: Train XGBoost model from [src/xgboost_model.py](src/xgboost_model.py), extract the trained `xgb.Booster`, compute SHAP values on the training set.
- **Key output**: SHAP beeswarm plot + bar importance plot

## Section 4: Calibration Curve (Probability Quality)

- **Tool**: sklearn `calibration_curve` + matplotlib
- **Purpose**: Are our predicted win probabilities accurate? If we say "70% chance," does that team actually win ~70% of the time historically? Poor calibration = overconfident or underconfident model.
- **Data**: Historical predictions from the optimizer's `_build_eval_games()` (1,070 games, 2008-2025). Predict with current weights, bin probabilities, compare to actual outcomes.
- **Key output**: Reliability diagram (predicted vs actual win rate) + Brier score

## Section 5: Parameter Distributions by Seed Tier

- **Tool**: seaborn `boxplot` or `violinplot`
- **Purpose**: For each parameter, do 1-seeds actually score higher than 16-seeds? Find parameters with poor separation (low signal) or inverted separation (actively hurting predictions).
- **Data**: 68 current teams grouped into tiers: elite (1-2 seeds), mid (3-6), low (7-10), longshot (11-16).
- **Key output**: Grid of boxplots, one per parameter, colored by seed tier

## Section 6: ORIGINAL vs CORE Weight Comparison

- **Tool**: matplotlib grouped bar chart
- **Purpose**: Side-by-side comparison of hand-tuned `ORIGINAL_WEIGHTS` vs optimizer-derived `CORE_WEIGHTS`. Highlights where the optimizer made the biggest moves and whether those moves are justified.
- **Data**: Both weight dicts from [src/weights.py](src/weights.py)

## Libraries Used

- **matplotlib + seaborn**: heatmaps, bar charts, boxplots, calibration curves (static, publication-quality)
- **shap**: XGBoost feature importance (the gold standard for ML explainability)
- **sklearn**: calibration curve utility
- No Plotly/Streamlit needed -- all static charts suitable for a notebook workflow
